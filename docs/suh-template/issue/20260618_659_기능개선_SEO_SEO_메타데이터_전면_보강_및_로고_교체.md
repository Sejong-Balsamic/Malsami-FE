📝 현재 문제점
---

- SEO 설정이 루트 `layout.tsx` 한 곳에만 최소한으로 존재한다.
- `metadataBase`가 없어 OpenGraph·canonical 상대경로가 절대경로로 변환되지 않는다.
- `sitemap` / `robots` / `manifest`가 전혀 없어 검색엔진 크롤링·PWA 지원이 부재하다.
- OG·favicon 이미지가 외부 무료 호스팅(`i.ibb.co`) URL에 의존해 호스팅 장애 시 깨진다.
- OG URL이 구 테스트 도메인(`test.sejong-malsami.co.kr`)으로 하드코딩되어 있다.
- 동적 상세 페이지(자료/질문/공지)가 전부 클라이언트 컴포넌트라 페이지별 메타데이터가 없다.
- 프로젝트 전반에 구 로고 자산(`logo.webp`, `old-logo.webp`, `deprecated/Logo.svg`)이 흩어져 있고 신규 로고로 통일되지 않았다.

🛠️ 해결 방안 / 제안 기능
---

- SEO 값(도메인·기본 title/description·OG 이미지)을 단일 소스(`src/lib/seo.ts`)로 모은다.
- 프로덕션 도메인을 `https://sejong-malsami.suhsaechan.kr`로 적용한다(환경변수 우선 + 코드 기본값 fallback).
- `layout.tsx`에 `metadataBase`, title template 패턴, OpenGraph, robots 설정을 보강하고 외부 `i.ibb.co` URL과 중복 `<head>` 메타 태그를 제거한다.
- `sitemap.ts`, `robots.ts`, `manifest.ts`를 신규 생성한다.
- 신규 로고 2종으로 전면 교체한다. 투명 배경 로고는 헤더 로고로, 검은 배경 로고는 파비콘·앱 아이콘·OG 이미지로 사용한다.
- 파비콘은 Next.js App Router 컨벤션(`app/icon.png`, `app/apple-icon.png`)으로 전환해 외부 URL 의존을 제거한다.
- 구 로고 자산(`logo.webp`, `old-logo.webp`, `deprecated/Logo.svg`)을 삭제한다.
- 동적 상세 페이지 SEO는 이번 범위에서 골격(패턴 + TODO 주석)만 잡고, 공개 API 확정 후 별도 작업으로 분리한다.
