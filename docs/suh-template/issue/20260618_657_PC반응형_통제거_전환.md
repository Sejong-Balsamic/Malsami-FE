## 📝 현재 문제점

운영 중인 세종말싸미 FE는 전역 레이아웃이 화면 중앙에 **고정 폭 컨테이너(모바일 640px / PC 960px)** 를 두고 모든 페이지·헤더·네비를 그 안에 가두는 구조였습니다. 이로 인해 다음 문제가 있었습니다.

- **PC 사용성 저하**: 넓은 화면에서도 가운데 좁은 영역만 사용하고 양옆이 빈 회색 배경으로 남아, "모바일 화면을 PC에 띄운" 인상을 벗어나지 못함
- **콘텐츠 폭 일률 적용**: 리스트·홈처럼 넓게 펼칠 콘텐츠와 상세·글쓰기처럼 읽기 좋게 좁혀야 할 콘텐츠가 동일한 폭으로 제한됨
- **게시판 카드 정렬 불균형**: PC 2열 그리드에서 카드 높이가 내용에 따라 들쭉날쭉
- **반응형 부재**: `lg:` 사용이 극소수에 불과해 PC 대응이 사실상 없었음

이슈 #655의 후속으로 분리해 두었던 "반응형 전환" 항목에 해당합니다.

## 🛠️ 해결 방안 / 제안 기능

중앙 고정 폭 컨테이너(통)를 완전히 제거하고, 화면 폭에 따라 레이아웃이 재배치되는 진짜 반응형으로 전환합니다. 모바일은 기존과 동일하게 유지하고 PC(lg 이상)에서만 레이아웃이 확장됩니다.

- **모바일(기본)**: 1열 풀폭 + 기존 하단 탭바 유지 (시각적 변화 없음)
- **PC(lg+)**: 상단 글로벌 네비 + 넓은 중앙 콘텐츠 영역

- [x] **PC 상단 글로벌 네비 신설**: 로고 + 메뉴(홈/자료/질문/마이), PC에서만 표시

- [x] **게시판 카드 PC 2열 그리드 높이 정렬**: 카드 높이를 셀에 맞춰 정렬해 좌우 균형 확보, 썸네일 PC 확대
