## 📝 현재 문제점

운영 중인 세종말싸미 FE(v0.1.39)는 빌드는 통과하지만, 누적된 스타일링 비일관성으로 인해 디자인 의도가 화면에 제대로 반영되지 않고 있습니다. 진단 결과 다음이 확인되었습니다.

- **죽은 폰트 클래스**: `font-pretendard-{medium,semibold,bold}` 121건(42개 파일)이 `tailwind.config`·`globals.css` 어디에도 정의되어 있지 않아 전부 무시됨 → 굵기/텍스트 계층이 적용되지 않음
- **픽셀 하드코딩**: `rounded-[8px]`·`gap-[4px]`·`ml-[8px]` 등 Tailwind 표준 스케일에 1:1 대응 가능한 임의값 픽셀이 다수 → 과도한 커스텀, 토큰 일관성 저하
- **정적 인라인 스타일**: `style={{ cursor: "pointer" }}` 등 클래스로 대체 가능한 정적 인라인 스타일 잔존
- **(후속) 반응형 부재**: `sm:/md:/lg:` 사용 6개 파일뿐, PC 대응 사실상 없음
- **(후속) 구조 비일관**: 타입 25개 파일 명명 난장판, 훅 2개 폴더 분산, `global/` 잡탕, Redux 슬라이스 네이밍 정리 필요

## 🛠️ 해결 방안 / 제안 기능

토큰/표준으로 수렴하는 위생 청산을 **위험도 낮은 순서로 단계별 PR**로 진행하며, 각 단계마다 `prettier → npm run build` 검증을 통과한 것만 반영합니다.

⚙️ 작업 내용
---
- [x] **죽은 폰트 청산**: `font-pretendard-*` → 프로젝트 표준 `font-suit-*`(globals.css 실존 유틸) 전수 치환 (42파일)
- [x] **픽셀 하드코딩 토큰화**: 표준 스케일 대응 px → Tailwind 표준 클래스 (`max-w-[640px]` 공인 컨테이너 표준은 보존)
- [x] **정적 인라인 스타일 정리**: `cursor: "pointer"` → `cursor-pointer` (동적 계산값 인라인은 보존)
- [ ] **(후속) 반응형 전환**: 시각 검증 필요 → 담당자 복귀 후 진행
- [ ] **(후속) 구조 재편**: 타입/훅/global/Redux 정돈 (import 대량 변경 → 신중 진행)

각 단계는 빌드 검증을 통과했으며, 디자인 변경이 시각적으로 검증되어야 하는 항목(반응형·구조 재편)은 안전을 위해 후속 작업으로 분리했습니다.
