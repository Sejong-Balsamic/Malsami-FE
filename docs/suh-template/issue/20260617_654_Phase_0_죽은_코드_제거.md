## 📝 현재 문제점

- 이 프로젝트에는 현재 선언만 되고 사용되지 않는 Redux 슬라이스 5개, 미사용 API 함수(BE에도 엔드포인트 없음), 그리고 전체 주석 처리된 API 파일이 존재합니다.
- 이러한 "죽은 코드(Dead Code)"는 이후 진행될 레거시 현대화 및 리팩토링 단계(API 표준화, 상태관리 재설계, UI 교체 등)에서 불필요한 인지 부하(Cognitive Load)를 발생시키고, 빌드 및 분석 시 노이즈를 초래합니다.
- 따라서 본격적인 로직 리팩토링 및 UI 수술 전에 가장 하단의 사용되지 않는 코드를 청소하여 깔끔한 기반을 구축할 필요가 있습니다.

## 🛠️ 해결 방안 / 제안 기능

- 사용처가 0건인 Redux 슬라이스 5개(`filterOptionsSlice`, `docHotDownFilterOptionsSlice`, `docMyFacultyFilterOptionsSlice`, `docRequestFilterOptionsSlice`, `facultySlice`)를 완전 삭제하고, `src/global/store/index.ts`에서 등록을 해제합니다.
- 사용처와 백엔드 엔드포인트가 모두 없는 `memberApi.getAllMemberPost` 함수(`/api/member/my-post`)를 제거합니다.
- 전체 주석 처리되어 있으며 import처가 0건인 `src/apis/landingApi.ts` 파일을 삭제합니다.
- 매 삭제 작업마다 `npm run build`를 통해 빌드가 정상적으로 완수되는지 확인하고, 개별 커밋으로 안전하게 적용합니다.
