# 고정 폭 컨테이너 제거 후 PC 진짜 반응형 전환

## 개요

전역 레이아웃이 화면 중앙에 고정 폭 컨테이너(모바일 640px / PC 960px)를 두고 모든 페이지를 그 안에 가두던 구조를 제거했습니다. 이제 화면 폭에 따라 레이아웃이 재배치되는 반응형으로 동작합니다. 모바일은 기존과 동일하게 1열 풀폭 + 하단 탭바를 유지하고, PC(lg 이상)에서는 상단 글로벌 네비와 넓은 콘텐츠 영역을 사용합니다. 콘텐츠는 성격별로 폭을 나눠(리스트·홈은 넓게, 상세·읽기 본문은 좁게) 배치합니다.

## 기능 흐름

```mermaid
flowchart TD
    A(["페이지 진입"]) --> B{"화면 폭 lg 이상?"}
    B -->|아니오 (모바일)| C["페이지별 고정 헤더 표시"]
    C --> D["1열 풀폭 콘텐츠"]
    D --> E["하단 탭바 (CommonNav)"]
    B -->|예 (PC)| F["상단 글로벌 네비 (GlobalTopNav)"]
    F --> G{"콘텐츠 성격?"}
    G -->|리스트/홈/마이/검색| H["content-wide 1200px"]
    G -->|상세/글쓰기/정책| I["content-narrow 720px"]
    H --> J["중앙 정렬 + 그리드 배치"]
    I --> J
    F -.페이지별 헤더 lg:hidden.-> J
```

## 변경 사항

### 전역 레이아웃 (통 제거)
- `src/app/ClientLayout.tsx`: 중앙 고정 폭 컨테이너와 바깥 회색 배경 제거. PC 상단 글로벌 네비 렌더, 콘텐츠에 PC 상단 네비 높이만큼 여백(`lg:pt-16`), 하단 탭바는 PC에서 숨김.
- `src/components/layout/GlobalTopNav.tsx`: PC 전용 상단 가로 네비 신설(로고 + 홈/자료/질문/마이 메뉴 + 인증 체크). `hidden lg:block`으로 PC에서만 표시.
- `src/components/layout/AppContainer.tsx`: `PageContainer`에 `width` prop(narrow/wide/legacy) 추가. `TopBarContainer`는 PC에서 숨김(`lg:hidden`) 처리.

### 헤더 PC 일원화
- `src/components/header/CommonHeader.tsx`, `LandingHeader.tsx`: 헤더 높이 스페이서를 모바일 전용(`lg:hidden`)으로. PC에서는 GlobalTopNav가 헤더를 대체하므로 페이지별 헤더가 사라짐(중복 제거).

### 콘텐츠 폭 토큰
- `tailwind.config.ts`: `maxWidth`에 `content-narrow`(720px), `content-wide`(1200px) 토큰 추가.
- 게시판 리스트·홈·마이페이지·검색·공지 리스트 페이지: 본문을 `PageContainer width="wide"`로.
- 상세·정책·도움말·공지 상세 페이지: `PageContainer width="narrow"`로. 각 페이지의 헤더 스페이서도 `lg:hidden` 적용.

### 게시판 카드 PC 그리드 정렬
- `src/components/documentMain/DocumentCard.tsx`, `questionMain/QuestionCard.tsx`: 카드 루트를 `flex h-full flex-col`로, 하단 정보(좋아요/조회수)를 `mt-auto`로 바닥 정렬. PC 2열 그리드에서 카드 높이가 달라도 좌우 정렬 유지. 썸네일 PC 확대(70→90px).
- `DocumentCardList.tsx`, `QuestionCardList.tsx`: 카드 래퍼에 `lg:h-full` 적용해 그리드 셀을 채우도록.

### 상세/마이페이지/랜딩 PC 레이아웃
- `questionDetail/QuestionDetail.tsx`, `AnswerSection.tsx`: 본문 읽기 폭 제한(`lg:max-w-[70ch]`), 첨부 이미지 PC 확대.
- `documentDetail/DocDetail.tsx`: 중복 폭 클래스 정리, 본문 읽기 폭 제한.
- `mypage/page.tsx`, `InfoList.tsx`, `Facility.tsx`: PC 2단 배치 및 메뉴 2열 그리드, 폰트 통일.
- `landing/WelcomeSection.tsx`, `LandingAllDocumentsSection.tsx`, `LandingAllQuestionsSection.tsx`, `MovingCard*.tsx`: 홈 인사말 좌우 배치, 전체 리스트 PC 2열 그리드, HOT 슬라이더 카드 PC 확대.
- `common/CommonNav.tsx`: `className` prop 추가(PC 숨김용).

## 주요 구현 내용

- **모바일 불변 보장**: 모든 변경은 `lg:` 분기 또는 모바일에서 동일하게 보이는 형태로만 적용. 통 제거로 바깥 회색 배경이 사라지지만 모바일은 원래 통이 화면을 꽉 채웠으므로 체감 변화 없음.
- **폭 토큰 단일 관리**: 폭 값을 `tailwind.config`의 토큰 한 곳에서 관리. `PageContainer`의 `width` prop으로 페이지가 폭 값을 직접 다루지 않고 성격(narrow/wide)만 선언.
- **헤더 이원화**: 모바일은 페이지별 헤더, PC는 GlobalTopNav. 둘은 `lg:hidden` / `lg:block`으로 상호 배타라 겹치지 않음.

## 주의사항

- 게시판 리스트는 현재 PC 2열 그리드. content-wide(1200px) 폭에서 3열로 늘릴지는 추후 시각 검증 후 결정.
- 일부 하단 고정 바(댓글 입력 등)의 PC 정렬 미세 조정은 후속 과제.
- 이슈 #655 후속 항목이며, 선행 토큰화 작업(컨테이너 폭 토큰 도입)과 이어진다.
